## 📖 코드 예시

### 기본 정렬 설정
```javascript
const headers = [
  // 텍스트 데이터 - 왼쪽 정렬
  { title: '이름', key: 'name', align: 'start' },
  { title: '부서', key: 'department', align: 'left' },

  // 날짜 데이터 - 왼쪽 정렬
  { title: '입사일', key: 'hireDate', align: 'start' },
  { title: '생년월일', key: 'birthDate', align: 'left' },

  // 숫자 데이터 - 오른쪽 정렬
  { title: '급여', key: 'salary', align: 'end' },
  { title: '나이', key: 'age', align: 'right' },

  // 상태 데이터 - 가운데 정렬
  { title: '상태', key: 'status', align: 'center' },
]
```

## 🎯 정렬 가이드라인

### 📊 데이터 타입별 정렬

| 데이터 타입 | 정렬 방향 | align 값 | 예시 |
|-------------|-----------|----------|------|
| **텍스트** | 왼쪽 | `start`, `left` | 이름, 주소, 설명 |
| **숫자** | 오른쪽 | `end`, `right` | 가격, 수량, 점수 |
| **날짜** | 왼쪽 | `start`, `left` | 2024-01-01, 생년월일 |
| **상태/아이콘** | 가운데 | `center` | 활성/비활성, 아이콘 |
| **액션 버튼** | 가운데 | `center` | 편집, 삭제 버튼 |

### 🔍 왜 이렇게 정렬할까요?

#### 숫자 오른쪽 정렬의 이유
- **자릿수 비교**: 큰 수와 작은 수를 쉽게 비교할 수 있습니다
- **계산 편의성**: 덧셈, 뺄셈 시 자릿수가 맞춰져 있어 계산하기 쉽습니다
- **시각적 일관성**: 소수점이나 콤마 위치가 일정합니다

#### 텍스트 왼쪽 정렬의 이유
- **읽기 자연스러움**: 한국어와 영어는 왼쪽에서 오른쪽으로 읽습니다
- **스캔 효율성**: 첫 글자부터 차례로 비교하며 찾기 쉽습니다
- **일관된 시작점**: 모든 텍스트가 같은 위치에서 시작합니다

#### 날짜 왼쪽 정렬의 이유
- **일관된 형식**: 날짜 형식(YYYY-MM-DD)이 일정한 길이를 가지므로 왼쪽 정렬이 자연스럽습니다
- **텍스트와의 일관성**: 날짜도 문자열 형태로 표시되므로 텍스트와 동일한 정렬을 사용합니다
- **시각적 흐름**: 연도부터 시작하는 날짜 형식의 읽기 흐름과 일치합니다

### ⚠️ 주의사항

#### 혼합 데이터 처리
```javascript
// ❌ 잘못된 예시 - 숫자와 텍스트가 섞인 경우
{ title: 'ID', key: 'id', align: 'start' } // "USER001", "2", "ADMIN123"

// ✅ 올바른 예시 - 데이터 타입에 따라 분리
{ title: '사용자 ID', key: 'userId', align: 'start' }    // "USER001", "ADMIN123"
{ title: '순번', key: 'sequence', align: 'end' }         // 1, 2, 3
```

#### 긴 텍스트 처리
```javascript
// 긴 텍스트도 왼쪽 정렬 유지
{ title: '설명', key: 'description', align: 'start', width: 300 }
```

### 🎨 시각적 일관성

- **같은 데이터 타입**은 항상 **같은 정렬** 사용
- **관련된 컬럼**들은 **논리적으로 그룹화**
- **액션 버튼**은 보통 **테이블 끝**에 배치하고 **가운데 정렬**
